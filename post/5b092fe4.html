HTTP 协议、作用

超文本传输协议、应用层

超文本：支持超越文本的内容，如图片、视频、音频等文件

传输：A B 两点之间传输数据

协议：规范

作用：在计算机之间进行【两点传输】【超文本】文件的协议

项目实现的Web功能：

  • 建立连接：通常基于 TCP 协议 ，建立两点之间连接后进行传输
  • 发送请求：Http 请求包括请求头、请求体
  • 处理请求：服务端收到请求后解析处理
  • 响应请求：发送响应状态码、响应头、响应体
  • 关闭连接

HTTP 常见状态码

口诀：一信 二成 三重 四五败

1xx：提示信息，协议处理的中间状态，还需要后续操作

2xx：

  • 200：成功状态码
  • 204：无Body版
  • 206：part版（返回的body数据不是资源的全部

3xx：

  • 301：永久重定向
  • 302：临时重定向
  • 304：不具有跳转意义，资源未修改，重定向到缓存

4xx：

  • 404：资源不存在或者未找到
  • 400：请求报文有错误
  • 401：请求需要认证
  • 403：禁止访问

5xx：（服务端

  • 500：笼统错误码
  • 502：服务器网关或者代理出现问题
  • 503：服务器正忙，无法处理请求
